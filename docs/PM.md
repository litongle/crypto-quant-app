## 二、迭代节奏

**2 周迭代**，每迭代包含：计划会议 → 日常开发（每日站会）→ 评审 → 回顾

### 每日站会（15 分钟）
1. 昨天完成了什么？
2. 今天计划做什么？
3. 有什么阻碍？

## 四、质量门

| 质量门 | 触发时机 | 通过条件 |
|--------|----------|---------|
| QG-1 | Phase 1 结束 | 架构文档完整，所有代理确认 |
| QG-2.x | 每个 Task 结束 | QA 验证 PASS |
| QG-3 | 集成测试完成 | 所有关键路径通过 |
| QG-4 | 部署完成 | 监控正常，健康检查通过 |

### 每个 Task 验收检查

```
□ 功能完整性：所有功能点已实现
□ 代码规范：符合 STANDARDS.md
□ 接口契约：API 符合文档定义
□ 无回归：未破坏已通过的其他模块
□ 安全检查：无敏感信息硬编码
□ 文档同步：相关文档已更新
```
