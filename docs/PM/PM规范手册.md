### 2.2 责任矩阵 (RACI)

```
                    PM  Arch  Front  Back  Quant  DevOps  QA  Integr
需求确认             A    C      I      I      I      I      I     I
架构设计             I    A/R    C      C      C      C      I     I
开发执行             I    C      A/R    A/R    A/R    A/R    I     I
代码审查             I    C      C      C      C      C      I     I
功能验证             I    I      I      I      I      I      A/R   I
集成测试             I    C      C      C      C      C      C     A/R
部署上线             A    C      I      I      I      A/R    I     C
```

- **R** = 负责 (Responsible) - 执行工作
- **A** = 问责 (Accountable) - 最终责任
- **C** = 咨询 (Consulted) - 提供意见
- **I** = 知会 (Informed) - 知晓进展

### 6.2 风险等级定义

```
风险评分 = 影响程度 × 发生概率

🔴 高风险 (评分 9-16): 需要立即关注，制定应急计划
🟠 中高风险 (评分 4-8): 需要定期监控，设置阈值告警
🟡 中风险 (评分 2-3): 记录在案，纳入观察清单
🟢 低风险 (评分 1): 可接受风险，无需特别行动
```

### 6.3 风险触发条件

当满足以下任一条件时，必须立即升级到 PM：

- 风险事件实际发生
- 风险概率显著增加
- 缓解措施无效
- 风险暴露超出预期阈值
