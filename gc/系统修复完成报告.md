## 3. 改进效果评估  

| 维度 | 改进前 | 改进后 |
|------|--------|--------|
| 任务重复执行 | Worker×2 导致订单 & Alert 重复 | 单一 Worker，路由唯一 |
| 日志可追溯性 | 零散 stdout，易丢失 | 按日文件 & trace_id，全链路检索 |
| 告警触达 | 数据库留痕，无即时通知 | 5 分钟内 IM 推送，严重级别电话/邮件 |
| 可观测性 | 无系统级指标 | API / Celery / DB / Redis / Node / Nginx 全覆盖 |
| 容器稳定性 | 无探针，异常难自愈 | 健康探针 + Restart Policy + 资源限额 |
| MTTR | ~2 h | 预计 < 15 min |
