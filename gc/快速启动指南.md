# RSI 分层极值追踪量化交易系统  
快速启动指南 (Quick Start Guide)

> 本指南面向想要**快速在本地体验**系统全部功能的开发者 / 量化爱好者，按照步骤操作即可在 5-10 分钟内运行完整的后台 API 与前端管理界面。  
> 操作系统以 **Windows 10/11** 与 **Debian/Ubuntu / macOS** 为例，其他平台步骤一致。

---

## 1. 克隆源码

```bash
git clone https://github.com/your-org/rsi-layered-tracker.git
cd rsi-layered-tracker
```

> Windows PowerShell 同样执行上述命令，或直接下载 ZIP 并解压。

---

## 2. 配置环境变量

1. 复制模板  

   ```bash
   cp .env.example .env           # Linux / macOS
   # 或
   copy .env.example .env         # Windows CMD
   ```

2. 使用编辑器打开 **.env**，根据自己情况填写：
   - OKX API Key / Secret / Passphrase（如只想本地演示，可填任意占位符）
   - Redis / PostgreSQL 连接信息  
     本地无数据库可保持默认，或稍后启用 Docker 版。
   - 运行端口：`WEB_PORT=8080`（后端） `server.port=3000`（前端）

保存退出。  

> ⚠️ **千万不要**提交包含真实密钥的 .env 文件到公共仓库！

---

## 3. 安装依赖

### 3.1 后端（Python ≥ 3.9 推荐 3.11+）

```bash
python -m pip install -r requirements-dev.txt
```

如需完整生产依赖（含 TimescaleDB / Celery 等）：

```bash
python -m pip install -r requirements.txt
```

### 3.2 前端（Node.js ≥ 18）

```bash
cd frontend
npm install
cd ..
```

---

## 4. 一键启动（Windows）

在项目根目录双击或命令行运行：

```cmd
start.bat
```

脚本会在两个新终端窗口分别启动：

- Backend → `http://localhost:8080/api/docs`
- Frontend → `http://localhost:3000`

按任意键脚本可自动关闭所有子进程。

---

## 5. 手动启动（Linux / macOS / WSL）

### 5.1 启动后端

```bash
python run_dev.py
```

首次启动会自动生成若干**模拟数据**（持仓 / 订单 / 策略等），便于前端演示。

### 5.2 启动前端

```bash
cd frontend
npm run dev
```

默认监听 `localhost:3000`，代理请求自动转发到 `http://localhost:8080`。

---

## 6. 访问系统

浏览器打开：

- Swagger API 文档：<http://localhost:8080/api/docs>
- 管理后台：<http://localhost:3000>

初始账号密码：`admin / admin123`  
（位于 `mock/login` 逻辑，可在 `frontend/src/views/login/index.vue` 修改）

---

## 7. 运行状态检测（可选）

项目根目录执行：

```bash
python check_status.py
```

脚本将检测后端端口 / API 及前端端口是否正常，并输出彩色概览。

---

## 8. Docker 生产部署（概要）

```bash
docker compose -f docker-compose.yml up -d
```

镜像包含：

- FastAPI + Uvicorn
- TimescaleDB
- Redis
- Celery Worker / Beat
- Vue3 静态文件

如需修改映射端口请编辑 `docker-compose.yml` 中对应条目。

---

## 9. 常见问题

| 现象 | 解决方案 |
|------|----------|
| `psycopg2` 编译失败 | 直接使用 `requirements-dev.txt` 或安装 `postgresql-client`、`libpq-dev` |
| 前端 `vite` 启动报缺失组件 | 运行 `npm install` 确保依赖完整；若仍缺包根据提示 `npm i xxx -D` |
| 端口被占用 | 修改 `.env` 中 `WEB_PORT` 或 `frontend/vite.config.ts` 的 `server.port` |
| API 403 或 401 | 确认已在登录页获取 JWT 并在请求头携带 `Authorization: Bearer <token>` |
